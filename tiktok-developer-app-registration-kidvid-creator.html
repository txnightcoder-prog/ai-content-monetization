5

Set Redirect URI & Run the OAuth Flow Once

This is the one-time step that produces your access token and refresh token. You only do this once — the backend auto-refreshes from then on.
Redirect URI to add in the portal
https://developers.tiktok.com/tools/redirect
Add this under Login Kit → Redirect domain / URI. TikTok's own redirect tool catches the auth code for you — no server needed.
OAuth URL to open in your browser (fill in your Client Key)
https://www.tiktok.com/v2/auth/authorize/?client_key=YOUR_CLIENT_KEY&scope=user.info.basic,video.list&response_type=code&redirect_uri=https://developers.tiktok.com/tools/redirect&state=kidvid123
Replace YOUR_CLIENT_KEY with your App Key from the portal. Open this URL in a browser while logged into TikTok. It will redirect to developers.tiktok.com/tools/redirect?code=XXXX — copy the code value.
Exchange the code for tokens (run in terminal / Postman)
curl -X POST https://open.tiktokapis.com/v2/oauth/token/ \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "client_key=YOUR_CLIENT_KEY" \
  -d "client_secret=YOUR_CLIENT_SECRET" \
  -d "code=YOUR_AUTH_CODE" \
  -d "grant_type=authorization_code" \
  -d "redirect_uri=https://developers.tiktok.com/tools/redirect"
The response JSON contains access_token (valid 24 h) and refresh_token (valid 365 days). Copy both.
6

Add Tokens to GitHub Secrets + .env

Never commit tokens to git. Add them as GitHub Secrets so the CI/CD pipeline injects them into the Azure Container App automatically.
GitHub Secrets to add
(Settings → Secrets and variables → Actions → New repository secret)
TIKTOK_CLIENT_KEY ← App Key from the portal
TIKTOK_CLIENT_SECRET ← App Secret from the portal
TIKTOK_ACCESS_TOKEN ← access_token from Step 5 curl response
TIKTOK_REFRESH_TOKEN ← refresh_token from Step 5 curl response
Local .env (for running the backend locally)
TIKTOK_CLIENT_KEY=paste_app_key_here
TIKTOK_CLIENT_SECRET=paste_app_secret_here
TIKTOK_ACCESS_TOKEN=paste_access_token_here
TIKTOK_REFRESH_TOKEN=paste_refresh_token_here
Once the GitHub Secrets are saved, push any commit to main and the pipeline will inject them into the live Azure container automatically. The backend auto-refreshes the access token every 24 h — you never need to touch it again.
  • Add all 4 secrets to GitHub Actions secrets
  • Push a commit to trigger the deploy (or use Actions → Run workflow)
  • Open 📡 Performance Monitor → 🔌 Connections tab in your app to verify TikTok shows ✅ Connected
  • Click ☁️ Sync — your TikTok views/likes/comments/shares will appear in the dashboard
!

Important Notes

Review / Sandbox mode
New TikTok apps start in Sandbox mode. In sandbox, the OAuth flow only works for the TikTok account that created the app — which is fine for your personal use case. You do not need to submit for TikTok's formal review to use the API on your own account.
Children's content (COPPA)
Because KidVid Creator produces content for children but the TikTok account owner is an adult creator, your app itself isn't directed at children. Set Is your app directed to children under 13? to No in the portal — the creator is the user, not the children who watch the videos.
Token lifetime
Access token: 24 hours — auto-refreshed by the backend on every sync.
Refresh token: 365 days — you'll get an email from TikTok 30 days before it expires. Re-run Step 5 at that point.
